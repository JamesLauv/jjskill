# Agent Usage Notes

- When a user provides project documents, meeting notes, customer requirements, sales/pre-sales material, technical evaluations, or internal communication drafts and asks to organize, analyze, turn into communication material, create decision artifacts, compare plans, or choose an output format, prefer `jjskill` as the router.
- If the user clearly asks for document-to-decision work, use `jjskill-doc-to-decision-artifacts` directly.
- Preserve the workflow's core pattern: restore source facts, interact to capture business judgment, confirm output format, then produce decision-ready artifacts.
